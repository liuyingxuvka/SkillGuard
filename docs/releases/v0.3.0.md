# SkillGuard v0.3.0

SkillGuard v0.3.0 adds a native one-skill-at-a-time portfolio calibration runtime on top of the executable-contract foundation introduced in v0.2.0.

## Highlights

- Keep the operational portfolio registry private while representing active owned/adopted targets, pending adoption, explicit exclusions, retired skills, system skills, and supporting repositories without silently changing scope.
- Register every SkillGuard behavior change and invalidate old current evidence under the new complete Guard fingerprint.
- Require a current non-empty capability inventory for each target. Representative jobs name covered capabilities and evidence refs, and their canonical map must equal the full-run receipt coverage fingerprint.
- Permit result reuse only when the exact Guard change is registered, non-broad, non-intersecting, and the source, contract, command, environment, and coverage identities remain unchanged from the registered full receipt.
- Serialize impact, reuse, and graduation writes with an exclusive recoverable local lock. Write ticket or receipt artifacts before the registry references them.
- Audit the next candidate before work begins and block it while any earlier active target is stale, pending, baseline-only, blocked, or revalidation-required.
- Accept repository, installed-skill, scripts, or package roots for Guard identity and show both expected and actual identity on mismatch.

## Upgrade

Use the staged whole-tree installer from the repository root. The stage path must end in `.codex/skills/skillguard`.

```powershell
python .agents/skills/skillguard/scripts/skillguard_install.py --canonical-skill-root .agents/skills/skillguard --stage-root <staged-codex-home>/.codex/skills/skillguard --codex-home <codex-home> --prepare
python .agents/skills/skillguard/scripts/skillguard_install.py --canonical-skill-root .agents/skills/skillguard --stage-root <staged-codex-home>/.codex/skills/skillguard --codex-home <codex-home> --activate
```

Activation preserves the previous active copy and restores it automatically if post-activation validation fails.
