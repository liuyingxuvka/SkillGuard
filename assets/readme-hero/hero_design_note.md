# SkillGuard README Hero Design Note

Project summary: SkillGuard is a local runtime-contract and maintenance framework for Codex skills.

Target users: maintainers who create, audit, upgrade, or release Codex skills and skill suites.

Core problem: skill work can skip phases, overclaim evidence, add parallel routes, or reuse stale reports unless the route, evidence, checks, and closure boundary are explicit.

Core workflow: skill files enter a work contract, SkillGuard binds checks to the target's native route when one exists, evidence gates verify current work, and closure either blocks or accepts the run.

Hero tagline: A local runtime-contract system for keeping Codex skills honest before, during, and after skill work.

Visual concept: a light technical workflow banner with five large stages connected by a rail: Skill files, Work contract, Native binding, Evidence gates, and Closure.

Image keywords: source files, route contract, native binding, evidence gates, closure boundary, local script evidence, no parallel route.
